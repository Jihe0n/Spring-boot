1. 제네릭
클래스를 만들때 데이터 즉, 내부의 타입을 정하지하고
인스턴스를 만들때 즉, 외부에서 선언할때 데이터 타입을 지정하는 것 
제네릭은 기본데이터 타입은 사용할 수 없음


2. wrapper 클래스 >> 기본 데이터타입은 
객체가 아니기 때문에 만약 객체로써 필요한 경우가 있을때
기본 데이터타입을 객체인것 처럼 만들수 있는 클래스
=> 객체로 포장(wrap) 하는 객체
int -> integer 
double -> Double
ex) Integer id = new Integer(1)  // 숫자 1을 객체로써 만들었음
